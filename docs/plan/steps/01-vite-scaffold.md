# Step 1 — Vite scaffold

Lay the frontend foundation. No new features; just migrate the existing
inline-HTML playground under a Vite project with hash routing and the
`#/interpreter` route hosting the current behavior unchanged.

## Deliverables

1. `web/` directory with Vite + TypeScript scaffolding.
2. Makefile targets: `web`, `web-dev`, `web-build`.
3. Migrated playground at `#/interpreter` — same four stage buttons,
   same seed/inputs controls, same output pane, same `/api/health`
   footer pill.
4. `web/dist/` (or equivalent) copies into `src/Ccgnf.Rest/wwwroot/`
   via `web-build`.
5. The old `wwwroot/index.html` is replaced by the Vite build output.

## Files to create

```
web/
  .gitignore                 (node_modules/, dist/)
  package.json               name: @ccgnf/web; scripts: dev, build
  tsconfig.json              strict; target ES2022
  vite.config.ts             port 5173; proxy /api → http://localhost:19397
  index.html                 <div id="app"></div>; /src/main.ts entry
  src/
    main.ts                  bootstrap: instantiate router, mount root
    router.ts                hash routing
    api/
      client.ts              fetch wrapper; typed POST / GET helpers
      dtos.ts                DiagnosticDto, ProjectRequest, RunRequest, …
    pages/
      interpreter/
        index.ts             migrated logic from wwwroot/index.html
        style.css
    shared/
      nav.ts                 top bar with one link (Interpreter) + health pill
      layout.css             grid shell shared across pages
```

## Files to modify

- `Makefile` — add `web`, `web-dev`, `web-build` targets.
- `src/Ccgnf.Rest/Ccgnf.Rest.csproj` — exclude `wwwroot/*` from git if we
  go with option 1 (build → copy), OR leave it for option 2 (commit).
  **Decision:** start by committing `wwwroot/`; simpler for CI. Revisit
  if the bundle grows unwieldy.
- `.gitignore` — add `web/node_modules/`, `web/dist/`.
- `src/Ccgnf.Rest/wwwroot/index.html` — delete (will be regenerated by
  the Vite build output).
- `tests/Ccgnf.Rest.Tests/EndpointsTests.cs::Root_ReturnsPlaygroundHtml` —
  update its content-assertion to match the Vite-built `index.html`
  (probably still contains the string "CCGNF Playground" in the `<title>`).

No framework deps in this step.

## Behavior parity

Feature list to port (identical behavior, not pixel-identical):

- Textarea for `.ccgnf` source.
- Seed + inputs inputs.
- Buttons: Health, Preprocess, Parse, AST, Validate, Run,
  Create session, List sessions.
- Output pane: last response (pretty JSON); ok/error pill.
- Health ping on load; port displayed in header.

## Tests

- `EndpointsTests.Root_ReturnsPlaygroundHtml` still passes after the
  Vite-built `index.html` is in place.
- No new tests required in this step; the existing REST integration
  tests already exercise every endpoint the playground calls.

## CI impact

- `make ci` stays dotnet-only. The frontend is built manually with
  `make web-build` before committing updated `wwwroot/` contents.
- A follow-up may add a GitHub Actions job that does `cd web && npm ci
  && npm run typecheck`.
