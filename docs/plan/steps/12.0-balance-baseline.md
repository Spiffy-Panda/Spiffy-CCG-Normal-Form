## Step 12.0 — Balance baseline

First step of the step-12 balance arc. This step produces a
**trustworthy snapshot** of where the system is today so every later
balance step has a fixed reference point to regress against. No
balance changes happen here — only measurement.

Read first: [10.2-long-term-ai-plan.md](10.2-long-term-ai-plan.md)
§"Decks × AI — affinity vs adaptation" and §10.2h (tournament
endpoint). The tournament GUI/endpoint already exists; step 12 uses
it as an observatory, not a feature to build.

## Scope — in

- Run two reference tournaments at the current (post-bump) defaults:
  - `ai-testing-data/PairCorrectly.tournament.json` — matched-pair
    round-robin across every archetype-bot pair.
  - `ai-testing-data/SimpleSweep.tournament.json` — mirror variant:
    every bot piloting the same deck, so deck noise is factored out.
- Save both results JSON + JSONL learning logs in `ai-testing-data/`
  and commit them as the **regression baseline**. Name them
  `<config>.baseline.results.json` so later sessions diff against a
  known filename.
- Write a one-page summary under
  [`docs/plan/balance/baseline-2026-04-20.md`](../balance/baseline-2026-04-20.md)
  (new folder) with: current win rates, draw rate, avg game length,
  leaderboard, and top three most-lopsided matchups.

## Scope — out

- Editing any weights, cards, or decks. This step is pure
  measurement; editing is step 12.1 onward.
- Streaming / SSE tournament runs. A blocking run is fine at the
  current match count.
- New engine / deck / AI content.

## Tests

- No new automated tests. The artifact is the checked-in results
  JSON; if later sessions change behaviour, the diff against the
  baseline *is* the test.

## Commit template

```
Step 12.0: balance baseline — reference tournament artifacts

Runs PairCorrectly + SimpleSweep at post-bump defaults (40 games,
6000 inputs, 150000 events). Writes baseline JSONs to
ai-testing-data/ and a human summary to docs/plan/balance/.

No code changes.
```

## Exit criteria

- Both result JSONs committed under `ai-testing-data/`.
- Baseline summary page committed under `docs/plan/balance/`.
- Devlog entry appended.
- **One of two states is true** at the end:
  - Draw rate is < 70% on at least one config (games are
    decidable → proceed to 12.1), or
  - Draw rate is ≥ 70% on both and we've captured a concrete
    hypothesis for why (typically "AI floor is bad — 12.1 will
    address it"). Either outcome is a legitimate exit.

## Size

≤ 0.5 day.
