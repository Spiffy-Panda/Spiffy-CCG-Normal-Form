## Step 12.2 — Engine sanity pass

Confirm the **rules themselves** permit decisive play at a
reasonable tempo before sinking days into card design. If the
engine is gated too hard, all downstream balance work is wasted
because no deck composition can reach lethal in the allowed input
budget.

### Status (2026-04-20)

**Audit complete. No engine knob shipped yet.** Follow-up is
**gated behind Step 12.3's card-authoring pass** — see §"Follow-up
sequencing" below.

## Scope — in

- **Minimum-turns-to-lethal analysis.** Hand-craft a "theoretical
  Ember aggro" deck — the most damage-dense configuration the
  current card pool supports. Pilot it with the Hellfire bot (or
  whatever is most aggressive after 12.1). Compute the median
  turn count at which Conduit lethal is reached. Target ≤ 8 turns.
  If > 8, the engine is gated too hard and knobs need touching.
  ✅ **Done.** Probe produced 2.5 % decisive rate at 428 avg engine
  steps — far from ≤ 8 turns.
- **Engine knob audit.** Enumerate what could be tuned to shorten
  games without breaking the rules spirit: Conduit starting
  Integrity (currently 7 × 3 per player), Clash damage scaling,
  Aether refresh rate, Arena activation cadence. Write each as a
  one-line hypothesis + where it's declared in `encoding/engine/`.
  Do **not** touch them yet. ✅ **Done** (4 knobs enumerated).
- **Draw-guard audit.** What *forces* a game to end? Conduit
  collapse is the primary path; Arsenal-empty is a fringe case.
  Note whether the engine needs a soft tiebreaker (banner lead,
  total integrity, aggressor bonus) for timed test runs where
  neither side lands lethal in the input cap. ✅ **Done** — rules
  have no turn limit and no tiebreaker; what the harness prints as
  "draws" is almost entirely cap-hit.
- **Deliverable:** a one-page note at
  [`docs/plan/balance/engine-audit-2026-04-20.md`](../balance/engine-audit-2026-04-20.md)
  listing the three or four concrete engine knobs we *could* touch
  if 12.3–12.6 can't produce decisive games. Queue them; don't
  commit them. ✅ **Done.**

## Scope — out

- Editing rules **during 12.2**. The audit is finished; any rule

  blocks on 12.3's first card-authoring pass completing.
- New mechanics. A soft tiebreaker is explicitly fine because it
  only applies to test-harness runs that otherwise would draw;
  the lived experience of the human game doesn't change.
- Running benchmarks against the tuned rules — that's 12.5.

### Why this order

1. **12.3 first**, because the 12.0/12.1 diagnosis explicitly routed
   the draw problem to card density. Pulling an engine knob without
   first testing the card fix would conflate two explanations and
   poison the baseline.
2. **Knob 3 before Knob 1**, because Knob 3 is a pure
   measurement-layer improvement (every future diff becomes more
   legible) and doesn't change game semantics. It's almost free.
3. **Knob 1 before Knob 2**, because Knob 1 is one line in
   `encoding/engine/09-clash.ccgnf` and doesn't require a design-doc
   update; Knob 2 bumps a rules invariant that `design/GameRules.md`
   encodes explicitly.
