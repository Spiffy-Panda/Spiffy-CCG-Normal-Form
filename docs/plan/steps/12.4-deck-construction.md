## Step 12.4 — Deck construction

The four reference decks in `encoding/decks/` should honestly
exploit the card pool cleaned up in 12.3. This step swaps
bad-fit cards out of each list, declares each deck's archetype
truthfully, and pins its matched AI profile.

Read first:
- [`encoding/decks/`](../../../encoding/decks/) — `*.deck.json`.
- The 12.3 card-pool summary at
  `docs/plan/balance/card-pool-2026-04-20.md`.
- [10.2-long-term-ai-plan.md](10.2-long-term-ai-plan.md) §"Decks × AI"
  — the affinity model (Option B). Decks declare `suggested_ai`;
  mismatched pairings are intentionally worse.

## Scope — in

- **Per-deck card-swap pass.** For each of the four reference decks
  (`bulwark-control`, `ember-aggro`, `hollow-disruption`,
  `tide-thorn-combo`), replace any card whose 12.3 tag doesn't
  match the archetype's target ratios. Keep curve and faction
  identity intact — do not convert bulwark into an aggro deck,
  just make its control plan actually have a finisher.
- **Archetype honesty.** Re-read each deck's `archetypes: [...]`
  tag and redeclare it if the card mix drifted. A bulwark deck
  with zero closers isn't "control" — it's either "prison" (and
  needs a prison win con) or mislabelled.
- **Matched AI declaration.** Set `suggested_ai` on every deck to
  the stable AI profile that piloted it best in the 12.1 bench.
  That same pairing becomes the default in the tournament GUI's
  "Suggest an AI" flow.
- **Mismatch penalty validation.** For each deck, write a small
  tournament config that runs it against *every* stable AI. The
  matched pairing should win by ≥ 15 pp on decisive games. If
  not, the 12.1 iteration point kicks in before this step's
  final commit.

## Scope — out

- New card authoring. If 12.4 discovers a gap that 12.3 missed,
  stop and go back to 12.3 — don't sneak a card in here.
- New AI profiles. If the pairings call for a new bot variant,
  that's 12.5 territory.
- Deck formats (draft / limited) — reference decks are all
  `constructed` for this arc.

## Tests

- No new xunit tests (decks are data). The acceptance is the
  bench run.
- Add / update `DeckCatalog` tests if a deck's shape changes enough
  to trip the existing preset tests.

## Commit template

Per-deck:

```
Step 12.4: deck — bulwark-control, finisher swap.

Replaces 3× HoldTheLine with 3× ReinforcedBreach (2-Aether Force-3
Maneuver). suggested_ai: stable/v2-fortress-updated.

Bench: matched-pair vs every other deck's matched AI, +X decisive
games, -Y draws at same seed as 12.0 baseline.
```

## Exit criteria

- Every reference deck has `archetypes` that match its actual
  plan and a `suggested_ai` that wins the matched-pair bench by
  ≥ 15 pp on decisive games.
- Tournament re-run at the 12.0 baseline seed shows each deck
  posting ≥ 30% decisive-game win rate against all others when
  piloted by its matched AI.
- The tournament matrix cell for "matched pair A vs matched pair
  B" is never 0% or 100% — polar cells are the input to 12.6.

## Size

~1 day, if 12.3 set us up well. Longer if card-pool gaps surface
and step-12.3 needs follow-up.
