## Step 12.3 — Card threat audit

Every card in `encoding/cards/` needs to earn its archetype, and
every archetype needs enough Conduit pressure to actually threaten
lethal. The 12.0 baseline identified that two of four reference
decks carry essentially zero Conduit-damage cards — that's a
card-pool deficiency, not an AI or rules problem.

### Status (2026-04-20)

**Audit complete. Authoring not yet started.** The audit is
exhaustive; the follow-up is substantial (15–19 new closers across
four factions). This step is now in its authoring phase, which
proceeds per-card per-commit, with bench numbers attached.

- Audit deliverable: [`docs/plan/balance/card-pool-2026-04-20.md`](../balance/card-pool-2026-04-20.md).
- Verdict: three mono-factions (BULWARK, HOLLOW, TIDE) have **zero**
  `closer` cards; THORN has 14 %. Only EMBER (83 %) hits target.
- See §"Authoring follow-up" below for the concrete card queue,
  per-faction design hooks, the card-cluster skill updates, and the
  hand-off to 12.4.

## Scope — in

- **Per-card tag pass.** Enumerate every card in
  `encoding/cards/*.ccgnf` and tag each one into exactly one of:
  - `closer` — pushes Conduit damage (direct-damage Maneuvers,
    Force-carrying Units that can Clash Conduits, Arena-pressure
    effects).
  - `setup` — enables closers (card draw, Aether acceleration,
    tutoring, Banner builders).
  - `disruption` — stalls the opponent (removal, bounce, taxes,
    counter effects, heal/Mend, Fortify-to-self).
  - `filler` — none of the above. A card tagged `filler` is a
    candidate for rework; the archetype can usually absorb one or
    two but not a deck's worth. ✅ **Done** (115 cards tagged).
- **Per-faction pool stats.** For each faction file, produce:
  `closer / setup / disruption / filler` counts, plus average cost
  per bucket. Target ratios (negotiable per archetype):
  - Aggro / tempo: ≥ 50% closer, ≤ 10% disruption.
  - Control / defensive: ≥ 30% closer, ≥ 30% disruption.
  - Combo: ≥ 30% setup, ≥ 20% closer (the combo payoff has to
    exist and be tag-visible). ✅ **Done.**
- **Closer authoring / retuning where gaps exist.** BULWARK and
  TIDE/THORN flagged in 12.0 as zero-closer. Either author new
  Conduit-damage cards or reshape existing ones so Clash damage
  from those factions can reach Conduits reliably. Every card
  edit is **one commit** so the bench trail stays per-card.
  **In progress** — see §"Authoring follow-up".

## Scope — out

- Deck construction. 12.4 picks the specific lists from this pool.
  When 12.4 opens, it blocks on this step's final authoring pass
  closing.
- New mechanics at the engine level (no new keywords). If an
  archetype needs a new keyword to express its closer, that's a
  12.2 follow-up commit, not this step.
- Retag changes that don't materially change function (e.g.
  renaming `disruption` to `disrupt`) — pure naming churn is out.

### SKILL.md update plan

The `card-cluster` skill at [`.claude/skills/card-cluster/SKILL.md`](../../../.claude/skills/card-cluster/SKILL.md)
has to learn about role buckets before the authoring queue can run
through it. Without this update, every cluster is still seeded by
faction × rarity × type × keyword only — with no awareness of the
closer gap — and the PRNG will keep producing setup / disruption /
filler cards that don't address the audit.

Needed edits, in order:

1. **Add role-bucket awareness** to the skill's core contract
   (§"Core contract"). New invariant: *every generated card carries a
   role tag (`closer` | `setup` | `disruption` | `filler`) in its
   block comment, and role selection is PRNG-weighted by the gap
   surfaced by the latest `card-pool-<date>.md` audit note.*

2. **New workflow step — "Role per slot"** inserted between the
   existing step 3 sub-sections "Faction per slot" and "Card type
   per slot". Weighting rule:
   - Read the target closer % per faction from
     `docs/plan/balance/card-pool-<latest>.md`.
   - For each slot, compute `deficit = max(0, target% − current%)`.
   - Role weight: `closer = 1 + 5*deficit`, `setup = 1`,
     `disruption = 1`, `filler = 0.3`. Normalise, then
     `random.choices(...)`.
   - Seed offset: `seed + 400` (slots 1-4 use `+100/+200/+300`; role
     takes `+400`).

3. **Per-role ability templates** added to §"Generate each card"
   (the existing step 4). Each role gets a family of ability
   patterns the LLM pulls from instead of free-styling:
   - `closer` → direct-Conduit-damage Maneuver, Force-3+ Unit with
     a trigger that reaches Conduit, Standard that deals 1 to
     target Conduit per turn.
   - `setup` → card draw, Aether accel, tutor, Banner builder,
     Sprawl/Rally generator.
   - `disruption` → removal, bounce, counter, Mend/heal,
     Fortify-grant-to-self.
   - `filler` → vanilla body, narrow-effect support. Strongly
     discouraged by the role-weight (0.3) unless PRNG forces it.

4. **Sync-check extension** (§"Sync check"): the role in the block
   comment must match the effect encoded in the `Card{...}`. A
   `closer` tag with no Conduit-damage line is a sync mismatch.

5. **DISTRIBUTION.md format update** — the regeneration script at
   `tools/update-card-distribution.py` grows a role-count column per
   faction. Two tables (existing: rarity × faction; new: role ×
   faction). The skill's step 10 ("finalize") re-runs the script so
   the role tally stays in sync. This ties into the 12.3 audit
   re-run cycle: the next audit date's `card-pool-<date>.md` reads
   from DISTRIBUTION.md's new role table.

6. **New "Gap-aware cluster" mode** — when the user runs
   `/card-cluster` without any filter, the skill defaults to
   *gap-first* generation: role PRNG weights are overridden to pull
   closer slots for factions with the largest deficit. The user
   can still override with explicit filters (e.g., "10 HOLLOW
   setups"), but the default serves the audit follow-up.

7. **Presentation update** — §"Present to the user" displays the
   role tag alongside rarity:
   ```
   **Cinderling** [C] *(EMBER, Unit, closer) — 1 — 2/1*
   ```

Each of the seven edits lands in the same commit as the SKILL.md
update; no reason to split them. The commit message names the step:
`Step 12.3: card-cluster skill — role-bucket awareness + gap-aware
default`.

### 12.4 coordination

The hand-off to [`12.4`](12.4-deck-construction.md) is clean when
this step's final re-bench satisfies the exit criteria below.
Specific contracts:

- **12.4 doesn't author new cards.** If 12.4 finds a closer gap
  mid-deck-construction, it *stops and returns here* — per the 12.4
  "Scope — out" rule. This step is the only place new cards land
  for the step-12 arc.
- **12.4 reads from DISTRIBUTION.md + `card-pool-<date>.md`, not
  from git history.** The final re-bench write is what 12.4 uses
  to pick its lists. Until 12.3 closes, 12.4 is blocked.
- **Deck archetype declarations may shift.** If authoring reveals
  that `bulwark-control` can't actually "control" without an
  `aggro` sub-pattern (because the new closers are aggressive), the
  `archetypes: [...]` tag in `encoding/decks/bulwark-control.deck.json`
  is 12.4's job to rewrite — **not 12.3's.** We declare the cards
  honestly here; 12.4 declares the decks honestly there.
- **Matched AI stays pinned per 12.1.** The bench numbers in this
  step are measured with the matched experimental bots from 12.1
  (`fortress`, `hellfire`, `reaper`, `wavebreaker`). 12.4's
  `suggested_ai` declarations follow whatever bot shows the best
  decisive WR on the post-authoring deck.
- **If card authoring shifts bench > 10 pp on a matched pair, the
  12.1 AI-floor iteration loop fires *before* 12.4 opens.** This
  is the same "iteration point" rule from 12.1 §"Iteration point";
  it applies to every card commit in this step.

## Exit criteria

- [x] Every card carries exactly one of the four tags; no untagged
      cards remain. (Delivered in
      `docs/plan/balance/card-pool-2026-04-20.md`.)
- [ ] Every archetype's ratio meets its target band, re-verified in
      a post-authoring `card-pool-<date>.md`.
- [ ] **Every archetype has at least one credible lethal line against
      an empty board in ≤ 6 turns** when piloted by its matched AI
      from 12.1. Verified by a targeted one-matchup tournament per
      archetype.
- [ ] Post-authoring re-run of `PairCorrectly.tournament.json` at
      baseline seed shows **draw rate ≤ 40 %.** If missed after the
      full 19-card queue lands, 12.2's knob sequence fires per
      [`12.2`](12.2-engine-sanity-pass.md) §"Follow-up sequencing".
- [ ] `card-cluster` SKILL.md updated with role-bucket awareness and
      gap-aware default.
- [ ] 12.4 unblocked — deck-construction step can consume the new
      pool.
