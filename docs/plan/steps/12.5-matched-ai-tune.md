## Step 12.5 — Matched-AI tune + mismatch validation

> **Status: open, next up (2026-04-20).** 12.3 / 12.4 / 12.2 closed
> above; the current PairCorrectly bench has TiThWave at 95.8 %
> raw WR — wavebreaker is over-calibrated and fortress is tied with
> wavebreaker on `bulwark-control` per SimpleSweep. Both need
> attention before 12.6 can open. See
> [`12.INDEX-2026-04-20.md`](12.INDEX-2026-04-20.md) §"Primary
> next step" for the concrete session plan.

After 12.3 fixed the card pool and 12.4 built decks that use it
honestly, the AIs promoted in 12.1 are almost certainly no longer
optimal — the scoring environment they were tuned for has shifted.
This step re-converges every matched (deck, AI) pair to a local
optimum and verifies the Option-B mismatch penalty is visible in
benches, not just on paper.

Read first:
- [10.2-long-term-ai-plan.md](10.2-long-term-ai-plan.md) §"Decks × AI"
  (Option B: affinity + mismatch).
- The 12.4 summary; the current matched pairings and `suggested_ai`
  declarations are the starting point.
- Each deck's matched AI `weights.json` — these are the files
  this step actually edits.

## Scope — in

- **Matched tune.** For each reference deck, re-run the 12.1 floor
  rule under the *current* card pool. The floor (conduit_softness
  ≥ threat_avoidance under pushing / lethal_check) is the minimum;
  above that, tune the `default` and `early_tempo` weights so the
  bot plays the archetype plan cleanly.
- **Promotion gate.** Each tuned AI must beat its previous stable
  version in a mirror tournament by ≥ 55% decisive-game win rate
  over 40 games. No promotion without the bench artifact.
- **Affinity declaration.** Edit each stable AI's
  `weights.json` so its `affinity.archetypes` and
  `pairs_well_with` honestly list what it was tuned on. A profile
  that only knows aggro doesn't claim `control`.
- **Mismatch validation.** Run the full `PairCorrectly.tournament.json`.
  For every deck, the matched pair must beat the three
  mismatched-AI-on-same-deck pairings by ≥ 15 pp on decisive
  games. If a mismatch pair wins, either the AI isn't matched (go
  back a few lines and re-tune) or the archetype tags are wrong
  (12.4 follow-up).

## Scope — out

- New considerations or BT edits. Only weight-table edits live here.
- Authoring new experimental bots from scratch — that's a separate
  arc when the Option-B set is saturated.
- Deck swaps. Freeze decks at the 12.4 state for this step's
  bench to be interpretable.

## Tests

- Mirror and matched/mismatched tournaments are the acceptance test.
- The benchmark harness (`make bench-ai` from 10.2i) runs
  automatically in the promotion commit; result JSONs go under
  `ai-testing-data/12.5-<profile>-<timestamp>.results.json`.

## Commit template

Per-profile:

```
Step 12.5: AI tune — stable/v3-fortress-for-updated-bulwark.

Bench vs v2-fortress-updated: 24W 12L 4D (60% WR, floor clears).
Affinity: archetypes=[control], pairs_well_with=[bulwark-control].
```

Aggregate at the end:

```
Step 12.5: matched-pair validation — every deck's matched AI beats
mismatched AIs by ≥ 15 pp on decisive games.

Bench JSONs under ai-testing-data/12.5-*.results.json.
```

## Exit criteria

- Every reference deck has a `stable/` AI profile that clears the
  floor, clears the mismatch bar, and has honest affinity.
- Tournament re-run at 12.0 baseline seed: the leaderboard is
  *tightened* (top ≈ bottom ± 15 pp) rather than polar.
- Every matched pair's decisive-game win rate is ≥ 15 pp above
  any mismatched pairing using the same deck.

## Size

~1 day, assuming decks and cards are frozen for the duration.
Longer if mismatches reveal archetype mislabelling in 12.4.
